View

The View handles presentation to the user (UI/UX).

- Controls what the user can see and interact with.

- Recieves data to display from the Model.

- Sends any user input to the Controller -- the View only handles display.

- Can sometimes recieve instructions directly from the Controller.
Example: Display some data in alphabetical order.
Reason: There's no need to interact with the Model, the data itself is not modified.
